# 文件状态追踪 + 去重逻辑优化 — 需求分析

## 背景

当前 FileMetadata 只有 READY / DELETED 两个状态，且前端知识库页面只显示文件名和大小，无法判断文件是否已完成向量化。去重逻辑只判断 MD5 是否存在，不判断处理进度。

## 核心业务逻辑

### 状态流转

```
UPLOADING   →  上传中（Redis 有 upload task，MySQL 尚未写入）
    │
    ▼
READY       →  上传完成（MinIO 合并完成，MySQL 已写入）
    │
    ▼
CHUNKED     →  文档解析+分块完成（DocumentConsumer 处理完）
    │
    ▼
EMBEDDED    →  向量化+ES 写入完成（VectorStoreConsumer 处理完）
```

状态不可逆，不可跳过中间状态。

### 去重规则（最终版）

| 场景 | 处理 |
|------|------|
| 同 MD5 + status=EMBEDDED（已完成全链路）| **秒传**，跳过所有步骤 |
| 同 MD5 + status<EMBEDDED（链路中断过）| **续传**：已完成的步骤跳过，从中断处继续 |
| 同 MD5 + 不同文件名 | **覆盖**：删旧 MinIO 对象 + MySQL 旧记录，走新上传 |
| 不同 MD5 | **新上传** |

### 删除逻辑

- 前端点删除 → `DELETE /api/storage/file/{md5}`
- 后端：删 MinIO 对象 + MySQL status=DELETED
- ES 中的 chunk 也一并删除（调 `DELETE /localrag-chunks/_doc/{chunkId}`）——后续可在 vector-store 模块补充

## 关键功能模块

### 1. FileMetadata 状态枚举扩展

```java
public enum Status {
    UPLOADING,   // 上传中（已在 Redis）
    READY,       // 上传完成（MinIO 合并成功）
    CHUNKED,     // 分块完成（DocumentConsumer 处理完）
    EMBEDDED,    // 向量化完成（ES 已写入）
    DELETED      // 已删除
}
```

### 2. 状态更新时机

| 状态变更 | 触发位置 | 模块 |
|----------|---------|------|
| → UPLOADING | `StorageController.initUpload()` | api |
| → READY | `StorageController.completeUpload()` / `directUpload()` | api |
| → CHUNKED | `DocumentConsumer.onMessage()` | document |
| → EMBEDDED | `VectorStoreConsumer.onMessage()` | vector-store |
| → DELETED | `StorageController.deleteFile()` | api |

### 3. 前端知识库页面优化

- 状态用不同颜色标签展示：蓝(UPLOADING)、黄(READY)、橙(CHUNKED)、绿(EMBEDDED)、红(DELETED)
- 删除按钮真正生效（调 DELETE API + 刷新列表）
- 显示时间 + 文件大小

## 边界条件

| 场景 | 处理 |
|------|------|
| 文件 EMBEDDED 后再次上传同名同内容 | 秒传 |
| 文件 READY 后被修改再上传 | MD5 变化 → 新上传，覆盖旧记录 |
| 删除 EMBEDDED 的文件 | MinIO + MySQL + ES 三处都删 |
| 删除 CHUNKED 但未 EMBEDDED 的文件 | MinIO + MySQL 删，ES 无数据直接过 |

## 改动范围

| 模块 | 文件 | 改动 |
|------|------|------|
| storage | FileMetadata.java | Status 枚举加 UPLOADING/CHUNKED/EMBEDDED |
| storage | FileMetadataRepository.java | 加 updateStatus(md5, status) 方法 |
| api | StorageController.java | 调整去重 + deleteFile 逻辑 |
| document | DocumentConsumer.java | 分块完成后更新 status=CHUNKED |
| vector-store | pom.xml | 加 storage 依赖 |
| vector-store | VectorStoreConsumer.java | ES 写入后更新 status=EMBEDDED |
| api | kb.html | 状态颜色标签 + 删除按钮修复 |
