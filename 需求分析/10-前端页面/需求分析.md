# 前端页面 — 需求分析

## 现状

共 5 个静态页面，各自独立维护 CSS，导致侧边栏样式不一致。

### 现有页面

| 页面 | 功能 | 侧边栏宽 | 品牌间距 | 导航结构 |
|------|------|---------|---------|---------|
| index.html | 控制台首页 | 200px | margin-bottom 28px | `<nav>` |
| upload.html | 分片/直传文件 | 200px | margin-bottom 28px | `<nav>` |
| kb.html | 知识库文件列表 | 200px | margin-bottom 28px | `<nav>` |
| chat.html | 对话助手 | **240px** | **border-bottom** | **`<div class="nav-top">`** |
| sessions.html | 历史会话管理 | 200px | border-bottom | `<nav>` |

### 不一致问题

1. **chat.html 侧边栏宽 240px**，其他页 200px——切换时视觉跳跃
2. **品牌区域** 3 种写法：`margin-bottom`、`border-bottom`、`padding` 各不同
3. **导航标签** 2 套：`<nav> a` 和 `.nav-top a`，padding/gap 不同
4. **字体图标写法** 2 种：chat 用 Unicode 直写 `&#128196;`，其他用 `<span class="ico-sm">`
5. **无共享 CSS 文件**——每页复制粘贴，改一处要改 5 个文件

## 目标结构

### 页面路由

```
/                    → index.html     控制台
/upload.html         → upload.html   分片/直传上传
/kb.html             → kb.html       知识库文件列表
/chat.html           → chat.html     对话助手（自动加载最近会话）
/sessions.html       → sessions.html  历史会话（搜索+筛选+管理）
```

### 侧边栏规范（全部 5 页统一）

```
侧边栏：200px，白底圆角卡片，左侧留白 16px 间距
  ├─ 品牌区：LocalRAG logo + 名称，底部 1px 分隔线
  ├─ 导航区（4 个平级入口）：
  │   ├─ 📄 文件上传
  │   ├─ 📚 知识库
  │   ├─ 💬 对话助手
  │   └─ 🕐 历史会话
  └─ （仅对话助手页） [+ 新对话] 按钮
```

### 主内容区规范

| 页面 | 内容 |
|------|------|
| index.html | 流水线状态 + ES/MinIO 连接信息卡片 |
| upload.html | 分片模式切换 + 文件选择 + 进度条 + 日志区 |
| kb.html | 刷新按钮 + 文件表格（文件名/MD5/大小/状态/时间/操作） |
| chat.html | 消息流 + 底部固定输入框（自动加载最近会话） |
| sessions.html | 搜索框 + 日期筛选 + 会话卡片列表（置顶/改名/删除） |

### 公共组件

| 组件 | 说明 | 使用页面 |
|------|------|---------|
| 自定义确认弹窗 | `showConfirm()` 替代浏览器 confirm | 全部 |
| 状态标签 | `.badge-uploading/ready/chunked/embedded` | kb.html |
| 消息气泡 | `.msg.user / .msg.assistant` | chat.html |

## 需要修正

1. **chat.html 侧边栏宽改为 200px**，与其他页一致
2. **品牌区统一为 `border-bottom + 无 margin`**（与 chat/sessions 对齐）
3. **导航标签统一样式**（gap/padding/font-size 一致）
4. **字体图标统一写法**（全部用 Unicode 直写，去掉 `<span class="ico-sm">`）
5. 考虑抽取一份 `common.css`？——当前阶段各页面独立样式差异较大，抽取成本 > 收益，暂不抽取
