# 提交描述模板

## 1. 完整模板

```
type: 简短说明本次改动

本次已解决：
- 
- 

本次已确认：
- 

当前仍未完全解决：
- 

说明：
本次提交属于阶段性改动，不代表相关功能已经最终闭环。
```

## 2. 小提交模板

```
type: 简短说明本次改动

本次已解决：
- 
```

## 3. 常用 type

| type | 用途 |
|------|------|
| feat | 新功能 |
| fix | 修复问题 |
| refactor | 重构整理 |
| docs | 文档更新 |
| test | 测试相关 |
| chore | 配置/脚本/环境调整 |

## 4. 写法要求

- 标题说清楚"这次主要改了什么"，不写"修复问题""完善功能"这种空话
- `本次已解决` 只写已经实际处理的内容
- `本次已确认` 只写已经运行或联调验证到的现象
- `当前仍未完全解决` 写遗留问题，避免把阶段性结果写成最终完成
- 小改动不用写"本次已确认"和"当前仍未完全解决"

## 5. 本项目示例

```
feat: storage 模块 — MinIO 分片上传 + 断点续传

本次已解决：
- MinIO multipart upload 分片上传（init/part/complete）
- Redis 记录上传进度，key=md5，支持断点续传
- 相同 MD5 秒传逻辑
- 上传进度查询接口

本次已确认：
- Docker 环境 MinIO/Redis 连通性正常
- 单分片 < 5MB 时走 direct upload
- Redis key TTL 24h 自动过期

当前仍未完全解决：
- Redis Keyspace Notification 触发 abort 未接入
- 定时任务清理僵尸 upload 未实现

说明：
本次提交属于阶段性改动，storage 模块上传链路已跑通，断点续传和秒传逻辑已就位。
```
